Good news: mount is fine and your E01 is there. The only issue now is dc3dd syntax.

It aborted because it thinks bs=4M is an “option”. That happens when dc3dd expects GNU-style bs=4M is NOT accepted on your build and wants bs=4096k or bs=4m (lowercase), or you accidentally pasted a weird character before bs.

Stop fighting it. Use a safe block size that dc3dd definitely accepts.

Run this (copy/paste)

mkdir -p ~/dc3dd_out
sudo dc3dd if=/mnt/ewf/ewf1 of=/home/vboxuser/dc3dd_out/sample.dd bs=4096k count=64 hash=sha256 log=/home/vboxuser/dc3dd_out/dc3dd.log

Then proof commands (separately):

ls -lh ~/dc3dd_out/sample.dd
tail -n 25 ~/dc3dd_out/dc3dd.log
sha256sum ~/dc3dd_out/sample.dd | tee ~/dc3dd_out/sample.dd.sha256
