What are sample.dd and sample.dd.sha256?

  • sample.dd = a raw disk image copy you created with dc3dd. It’s a copy of data, not the original USB.
  • sample.dd.sha256 = the SHA-256 hash of that image, so you can prove integrity (the image wasn’t altered).

The connection to “USB” is: your evidence is usb_a.E01 (a forensic image of a USB). You mounted it to get a raw stream (/mnt/ewf/ewf1) and then dc3dd produced a raw .dd copy from that data stream. That’s why it’s related.

The Windows tools (lightweight, CLI, no Autopsy)

1) EvtxECmd (Windows Event Log parser)

Parses .evtx event logs (Security/Application/System) and exports to CSV/JSON/XML.

  • -f = process one .evtx file
  • -d = process a directory of .evtx files
  • --csv + --csvf = export CSV to a folder with a chosen filename
  • --inc = include only specific Event IDs (e.g., 4624 logons)

2) MFTECmd (NTFS $MFT parser)

Parses the NTFS $MFT (Master File Table) and exports CSV/JSON/bodyfile timelines. Since your CW only needs proof for the non-evaluated tool, you can keep MFTECmd as “proof it runs.”

How to run them (exact steps)

A) Put both tools in one folder: C:\Tools\EZ\

B) Open PowerShell in that folder: Shift + Right-click → Open PowerShell window here

C) “Proof it runs” commands:

.\EvtxECmd.exe -h
.\MFTECmd.exe -h

Phase 4 Windows — 3 examples using EvtxECmd

Example W1 — Parse ONE log file (single-file mode)

Example W2 — Parse a DIRECTORY of logs (bulk mode)

Example W3 — Filter to specific Event IDs

.\EvtxECmd.exe -f "C:\Windows\System32\winevt\Logs\Security.evtx" --inc 4624,4625 --csv "C:\F20FO_CW1\Phase4\out" --csvf LogonsOnly.csv
