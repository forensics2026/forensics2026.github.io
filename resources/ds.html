F20FO CW1 Context Pack (2025–26)

According to the F20FO CW1 coursework sheet, your job is to build a specific prototype DF lab with strict deliverables. Use this as your single source of truth.

CW1 Non-Negotiable Requirements

What the coursework actually requires

  • Build a prototype virtual DF lab with two VMs: one Windows + one Linux (no security-suites like Kali).
  • Install at least two distinct DF tools on each VM (4 total).
  • Evaluate 2 tools total (1 Windows, 1 Linux) using literature and practical findings.
  • Provide 3 practical examples per evaluated tool (6 total).
  • Evidence resources from labs can be used for examples.

Video + Report Rules

Video: Max 5 minutes. >5 mins = Zero marks.

Report: 1500 words ±10%. No first-person language. Must include diagrams and references.

The "No Lab Tools" Rule

It is not a hard ban, but the rubric flags "All tools from labs" as weak. To hit the top-band marks, select two tools not covered in lab sessions for your deep evaluation.

Phase 1 Summary: VM & Networking

Goal

Proof of communication (ping) between Windows and Debian VMs without using security-specific distros.

Verified Configuration

# Check OS and Disk Install
cat /etc/os-release
df -h /

# Verify Network (Target: 10.0.2.15/24)
ip a

Networking Setup (NAT Network)

  1. Create Network: VirtualBox Tools > Network Manager > NAT Networks.
  2. Name: ForensicsNet | Prefix: 10.0.2.0/24.
  3. Apply: Settings > Network > Attached to: NAT Network.

Verification Commands

Windows (Allow Ping):

netsh advfirewall firewall add rule name="Allow ICMPv4" protocol=icmpv4:8,any dir=in action=allow

Debian (Test Connection):

ping -c 4 [WINDOWS_IP]

VM Housekeeping

  • Shutdown: Use normal shutdown. Take one snapshot only when Phase 2 tools are stable.
  • ISO: Disconnect the Debian ISO from storage settings to prevent rebooting into the installer.
  • Windows VM: The MSEdge - Win10.ova is acceptable. Import via File > Import Appliance.

Phase 2: Tool Installation Objective

Install 4 tools, select 2 for deep evaluation, and ensure they are not "lab tools" if aiming for high marks.

# Commands you will need frequently:
sudo apt update
ipconfig / ip a
ping [target_ip]
