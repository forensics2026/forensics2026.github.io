How we set up networking so the VMs can ping each other (Phase 1 core proof)

Why this matters

CW explicitly requires the two VMs communicate.
The cleanest VirtualBox approach is NAT Network (not Host-Only).

VirtualBox steps (GUI clicks)

  1. VirtualBox Manager → File → Tools → Network Manager
  2. Go to NAT Networks tab (NOT “Host-only Networks”)
  3. Click Create and set:
    • Name: ForensicsNet
    • IPv4 Prefix: 10.0.2.0/24
    • DHCP: Enabled
  4. For each VM (Windows + Debian):
    • Right-click VM → Settings → Network → Adapter 1
    • Attached to: NAT Network
    • Name: ForensicsNet
    • OK

Ping verification commands (what you show in the video + screenshots)

On Windows VM (CMD):

bat

If Windows blocks ping, allow ICMP once (Admin CMD):

Back on Windows (CMD):

VM housekeeping decisions we made (so you don’t screw yourself later)

Saved State vs Snapshot (your question)

  • Saved State = VirtualBox writes RAM state to disk and resumes later. It can cause weirdness and isn’t “clean” for demos.
  • Snapshot = checkpoint you can revert to (fine, but don’t spam them).

Recommendation for CW:
Use normal shutdown for daily use. Take one snapshot only when Phase 2 is stable (tools installed + working), so you can revert if you break stuff.

“Where is the saved state stored?”

Typically in the VM folder with files like .sav / snapshot folders. In VirtualBox Manager you can right-click the VM → Show in Explorer to open the folder (you already saw this option).

“Delete the ISO?”

Yes — after Debian is installed:

  • VM → Settings → Storage
  • Under Optical Drive / Controller IDE (or SATA Optical), ensure it shows Empty
  • That prevents booting back into installer.
