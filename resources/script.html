A. VirtualBox Setup

B. Debian VM ‚Äî Get Everything Ready

  • Start Debian VM, log in
  • Open a terminal (right-click desktop ‚Üí Terminal, or find it in applications)

Install ewf-tools (needed to mount .E01 files):

sudo apt update
sudo apt install ewf-tools -y

Install foremost:

sudo apt install foremost -y

Copy usb_a.E01 to the VM (drag-and-drop, shared folder, or USB):

  • Put it somewhere simple like: /home/YOUR_USERNAME/evidence/usb_a.E01
  • If you can't transfer files, download it from Canvas inside the VM's browser

Mount the evidence image (do this NOW, before recording):

sudo mkdir -p /mnt/ewf
sudo ewfmount /home/YOUR_USERNAME/evidence/usb_a.E01 /mnt/ewf
  • You should now see a file at /mnt/ewf/ewf1 ‚Äî this is the raw disk stream
  • Test it: ls -la /mnt/ewf/ ‚Äî you should see ewf1

Create the custom foremost config (for Example 3):

cp /etc/foremost.conf ~/evidence/custom_foremost.conf

Now edit it to keep ONLY image types:

nano ~/evidence/custom_foremost.conf
  • Scroll down. You'll see lines for jpg, gif, png, bmp, etc.
  • Delete or comment out (put # in front of) EVERY line that is NOT an image type
  • Keep these lines uncommented: jpg, gif, png, bmp
  • Comment out everything else: doc, xls, pdf, zip, avi, mov, exe, htm, etc.
  • Save: press Ctrl+X, then Y, then Enter

Clean up any old foremost output:

rm -rf ~/evidence/output_default
rm -rf ~/evidence/output_custom

Verify foremost works (quick test run ‚Äî delete output after):

foremost -V

Should print version number. If it says "command not found", go back and install it.

Shut down the Debian VM:

sudo shutdown now

C. Windows VM ‚Äî Get Everything Ready

  • Start Windows 10 VM, log in
  • Make sure EvtxECmd is ready:
    • Open File Explorer
    • Navigate to wherever you put the Zimmerman tools (e.g., C:\Tools\ZimmermanTools\)
    • Find EvtxECmd.exe ‚Äî note the FULL path (e.g., C:\Tools\ZimmermanTools\EvtxECmd\EvtxECmd.exe)

Test EvtxECmd works:

  • Open Command Prompt (search "cmd" in Start menu, right-click ‚Üí Run as Administrator)
  • Type:
C:\Tools\ZimmermanTools\EvtxECmd\EvtxECmd.exe --help

(Use YOUR actual path if different)

  • Should print help text. If it says "not recognized" or gives a .NET error, fix the path or install .NET 6 Runtime.

Create output folder:

mkdir C:\Evidence\EvtxOutput

Copy the event logs to a working folder (so you don't touch originals):

mkdir C:\Evidence\EventLogs
copy C:\Windows\System32\winevt\Logs\Security.evtx C:\Evidence\EventLogs\
copy C:\Windows\System32\winevt\Logs\System.evtx C:\Evidence\EventLogs\
copy C:\Windows\System32\winevt\Logs\Application.evtx C:\Evidence\EventLogs\

(You need the admin Command Prompt for this)

Shut down the Windows VM:

shutdown /s /t 0

"foremost says output directory already exists"

Then run foremost again.

"EvtxECmd gives a .NET error"

Download and install .NET 6 Desktop Runtime from:
https://dotnet.microsoft.com/en-us/download/dotnet/6.0
Pick the "x64" Windows installer under ".NET Desktop Runtime". Install, restart Command Prompt, try again.

"EvtxECmd says Access Denied on the .evtx file"

You need an Administrator Command Prompt:

  • Click Start
  • Type cmd
  • Right-click "Command Prompt"
  • Click "Run as administrator"
  • Click Yes
